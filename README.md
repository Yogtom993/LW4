# Лабораторна робота №4

## Тема
Bootstrap 4.

## Опис
Сторінка TechStore створена з використанням Bootstrap Grid. Реалізовано варіант №5: три ряди блоків, адаптивність для md, sm та xs, відступи, зсуви колонок і власні стилі блоків.

```text
├── css/
│   ├── bootstrap.min.css
│   └── style.css

└── img/
```

## Як перевірити адаптивність

1. Відкрити файл `index.html`.
2. Натиснути кнопки `Desktop`, `Tablet`, `SM`, `Mobile`.
3. Основний сайт відкривається у вікні `iframe`.
4. Ширина `iframe` змінюється, тому Bootstrap перебудовує блоки за своїми breakpoint-ами.

Файл `site.html` — це сама сторінка лабораторної роботи без панелі перевірки.

На xs-екрані у третьому ряду замість великого блоку відображається порожня область без тексту, як вимагає варіант.


## Перевірка breakpoint-ів

Для перевірки `md` використовується кнопка `Tablet / md 800px`, а не 768px.  
Це зроблено тому, що через рамку, scrollbar або iframe фактична ширина області перегляду може бути трохи меншою за 768px.  
При 800px Bootstrap 4 гарантовано сприймає сторінку як `md`.

- `Desktop 1200px` — повний вигляд.
- `Tablet / md 800px` — перший ряд має два блоки по 5 колонок, парні блоки другого ряду ще видимі.
- `SM 620px` — парні блоки другого ряду зникають.
- `Mobile / xs 375px` — у третьому ряду замість великого блоку показується порожній простір.


## Виправлення для xs

Великий блок третього ряду приховується тільки на `xs`-екрані.  
Для цього використано логіку Bootstrap 4:

- великий блок: `d-none d-sm-block` — прихований лише до 576px;
- порожній блок: `d-block d-sm-none` — показується лише на xs.

Тому на `SM 620px` великий блок уже має бути видимий, а на `Mobile / xs 375px` замість нього показується порожній простір.


## Перевірка третього ряду

У третьому ряду великий блок тепер не приховується класами Bootstrap напряму.  
Він має клас `xs-big-block`, який приховується тільки через CSS при ширині до `575.98px`.

Перевірка:
- `SM 620px` — великий блок видно;
- `Mobile / xs 375px` — великий блок зникає, замість нього видно порожній простір.
