# Лабораторна робота №4

## Тема

Bootstrap 4.

## Варіант

Варіант №5.

## Назва проєкту

**TechStore — каталог продукції компанії**

## Мета роботи

Вивчити фреймворк Bootstrap 4 та набути практичних навичок створення адаптивних вебсторінок.
Навчитися використовувати Bootstrap Grid System, класи колонок, відступів, зсувів, а також класи приховування та відображення елементів залежно від розміру екрана.

## Опис проєкту

У межах лабораторної роботи створено сторінку **TechStore** з використанням Bootstrap 4.

Сторінка демонструє адаптивну сітку відповідно до варіанту №5.
На сайті є три ряди блоків, які змінюють своє відображення залежно від ширини екрана.

Для зручної перевірки адаптивності створено окрему сторінку `index.html`, де є кнопки зміни ширини області перегляду:

* `Desktop 1200px`;
* `Tablet / md 800px`;
* `SM 620px`;
* `Mobile / xs 375px`.

Основна сторінка лабораторної роботи знаходиться у файлі `site.html`.

## Виконані завдання за методичкою

### Завдання №1

**Створити HTML-сторінку з використанням фреймворку Bootstrap відповідно до варіанту.**

Було створено сторінку `site.html`, у якій використано Bootstrap 4.
Bootstrap підключено локально через файл:

```html
<link rel="stylesheet" href="css/bootstrap.min.css">
```

Також підключено власний файл стилів:

```html
<link rel="stylesheet" href="css/style.css">
```

### Завдання №2

**Блокам кожного ряду задати власний стиль: рамки, відступи, колір тощо.**

Для блоків створено власні CSS-класи.
Блоки мають фон, рамки, внутрішні відступи, заокруглення, тіні та різне кольорове оформлення.

### Завдання №3

**Використовувати вбудовані класи відступів і зсувів колонок.**

У роботі використано Bootstrap-класи:

```text
container
row
col-12
col-sm-6
col-md-3
col-md-5
col-md-6
offset-md-1
mb-3
mb-4
p-4
py-4
text-center
d-sm-none
d-md-block
```

Ці класи відповідають за сітку сторінки, відступи, вирівнювання, зсуви колонок і адаптивне відображення елементів.

### Завдання №4

**Створити 3 ряди: у першому — 2 блоки, у другому — 4 блоки, у третьому — один великий блок на половину екрана і ще 4 блоки на другу половину.**

На сторінці реалізовано три ряди:

1. **Перший ряд** — 2 блоки:

   * ноутбуки;
   * смартфони.

2. **Другий ряд** — 4 блоки:

   * блок 1;
   * блок 2;
   * блок 3;
   * блок 4.

3. **Третій ряд** — один великий блок на половину екрана та 4 маленькі блоки на другу половину.

### Завдання №5

**Усі блоки повинні трансформуватися залежно від розміру екрана.**

Для адаптивності використано Bootstrap Grid System.
Блоки змінюють ширину та розташування залежно від розміру екрана.

### Завдання №6

**При зменшенні до `md` у першому ряду — 2 блоки по 5 колонок.**

У першому ряду використано класи:

```html
col-md-5
offset-md-1
```

Завдяки цьому на екранах `md` два блоки займають по 5 колонок.

### Завдання №7

**При зменшенні до `sm` у другому ряду — зникають парні блоки.**

У другому ряду парні блоки приховуються на малих екранах.
Для цього використано адаптивне відображення через Bootstrap-класи.

На `SM 620px` перевіряється поведінка малого екрана: парні блоки другого ряду зникають.

### Завдання №8

**При зменшенні до `xs` у третьому ряду — замість першого блоку з’являється порожній простір.**

У третьому ряду великий блок приховується тільки на `xs`.
Для цього використано клас `xs-big-block` і CSS-правило:

```css
@media (max-width: 575.98px) {
  .xs-big-block {
    display: none !important;
  }
}
```

Замість великого блоку на `xs` відображається порожній простір.

## Опис файлів

* `index.html` — сторінка для перевірки адаптивності з кнопками зміни ширини екрана;
* `site.html` — основна сторінка лабораторної роботи Bootstrap 4;
* `style.css` — додатковий файл стилів для сторінки перевірки;
* `README.md` — опис лабораторної роботи;
* `css/bootstrap.min.css` — локально підключений Bootstrap 4;
* `css/style.css` — основні стилі сторінки лабораторної роботи;
* `js/bootstrap.bundle.min.js` — файл Bootstrap JavaScript;
* `img/` — папка із зображеннями для сайту.

## Як запустити проєкт

1. Завантажити або клонувати репозиторій.
2. Відкрити папку проєкту.
3. Запустити файл `index.html` у браузері.
4. Натискати кнопки перевірки адаптивності:

   * `Desktop 1200px`;
   * `Tablet / md 800px`;
   * `SM 620px`;
   * `Mobile / xs 375px`.

## Як перевірити адаптивність

1. Відкрити `index.html`.
2. Натиснути **Desktop 1200px** — має відображатися повна структура сторінки.
3. Натиснути **Tablet / md 800px** — у першому ряду два блоки мають бути по 5 колонок.
4. Натиснути **SM 620px** — у другому ряду мають зникнути парні блоки.
5. Натиснути **Mobile / xs 375px** — у третьому ряду великий блок має зникнути, а замість нього має з’явитися порожній простір.

## Файли, які потрібно завантажити в репозиторій

У репозиторій потрібно завантажити не архів, а всі файли та папки з проєкту:

```text
index.html
site.html
style.css
README.md
css/
js/
img/
```

Без папок `css/`, `js/` та `img/` сторінка може відображатися неправильно.
